# Git 操作规范

本 skill 统一管理以下 7 个项目的 git 与云效（yunxiao）操作。所有分支命名、创建流程、云效仓库映射必须按本规范执行。

## 项目范围（唯一权威列表）

| 项目      | 说明              |
|-----------|-------------------|
| fbi       | 商业智能化分析系统（**默认项目**） |
| bi-common | FBI & HBI 基础公用 |
| ard-api   | 接口服务系统       |
| ard-etl   | 数据清洗系统       |
| message   | KIT 端 API 系统    |
| nl-data   | 实时数据系统       |
| report    | 客户端 API 系统    |

> 云效仓库 ID 等信息见 `reference/yunxiao.md`（备用，当前不使用）。

## 项目选择规则

| 用户输入                 | 实际操作范围                |
|--------------------------|---------------------------|
| 未指定项目               | 仅 `fbi`                  |
| 指定单个项目（如 `ard-etl`）| 该项目                    |
| 指定多个（如 `fbi, message`）| 列出的项目，逗号或顿号分隔  |
| 「全部」                  | 上表全部 7 个项目          |

> 「全部」是**约定关键字**，专指上表 7 个项目，不要扩展到其他仓库。

## 本地项目路径

所有项目统一存放在本机 `~/sites/` 目录下：

```
~/sites/fbi
~/sites/bi-common
~/sites/ard-api
~/sites/ard-etl
~/sites/message
~/sites/nl-data
~/sites/report
```

> **默认执行方式**：分支创建 = **本地 git 操作后 push 远端**。云效 MCP 仅用于查询分支/MR 等只读场景，不用于创建分支。

## 触发后必读

| 操作意图                    | 必读文件                |
|-----------------------------|-------------------------|
| 创建/切换分支、分支命名校验   | `reference/branch.md`   |

> `reference/yunxiao.md` 为**备用文档**，当前不使用，仅在用户明确要求走云效 MCP 时才读取。

## 强制行为

1. **任何分支创建前**，必须先与用户确认生成的分支名（按 `branch.md` 规范自动生成）
2. **不符合命名规范的分支请求** → 提示用户并拒绝执行
3. **多项目操作** → 逐项目串行执行并汇报每个仓库的结果，不要静默失败
