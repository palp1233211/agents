## 调用约定

1. **每次调用必须显式传入** `organizationId` + `repositoryId`，不要从上下文猜测
2. **批量操作（"全部"）** → 按上表顺序串行调用，每个仓库独立报告结果
3. **失败处理** → 单个仓库失败立即停下并报告，不要继续静默执行
4. **只读操作**（查询分支、查 MR）可以并行；**写操作**（建分支、建 MR）必须串行

## 分支命名联动

通过云效创建分支时，分支名仍按 `reference/branch.md` 规范生成，源分支固定为 `master`。

## 操作清单

| 意图               | 调用方向                                |
|--------------------|----------------------------------------|
| 创建远端分支        | yunxiao MCP `create_branch` + 上述参数  |
| 查询分支是否存在    | yunxiao MCP `list_branches` 后过滤匹配  |
| 创建 MR            | 必须先确认源/目标分支均存在再发起        |

> 具体 MCP 工具名以当前已注册的云效 MCP server 暴露的工具为准；调用前用工具列表确认实际命名。
