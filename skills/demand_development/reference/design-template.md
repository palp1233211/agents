## 开发方案模板（Step 1）

> 根据 Step 0 的拆解生成可落地的开发方案

### Git需求分支创建
按 [git规范](./git.md) 创建并切入需求分支

### 方案必须包含项

```markdown
## 1. 开发分支
分支名：`上面创建的分支名`

## 2. 数据变更
- 表/字段/索引/约束变更
- 迁移策略（含回滚/兼容方案）

## 3. 影响面分析

> **使用 Explore subagent 完成影响面探索**，不在主上下文直接读取项目代码：
    > ```
    > Agent(subagent_type="Explore", prompt="探索 {需求编号} 影响面：
    > 1. 找出所有涉及修改的文件（Controller/Task/BLL/Model）
    > 2. 找出调用链：哪些上游调用了待修改的方法
    > 3. 找出数据库表结构（如有变更）
    > 4. 返回：文件路径 + 行号 + 一句话描述，不贴原始代码")
    > ```
> 将 Explore 返回的摘要填入下表。

| 影响类型 | 具体项 | 兼容策略 |
|----------|--------|----------|
| 接口 | /api/xxx | 版本兼容 |
| 页面 | XXX页面 | 前端适配 |
| 定时任务 | task_xxx | 时间窗口 |
| 消息流 | queue_xxx | 消费端适配 |

### ⚠️ 公共 Task / Controller 强制检查

扫描所有变更文件，凡 Task 在 `app/tasks/` 或 Controller 在 `app/controllers/`（非 `app/modules/{Country}/` 子目录），且需求有国家/业务范围限制，**必须在方案中提醒用户确认**，并给出两个选项：

> **`{文件路径}` 在公共目录，其他国家可能也在使用。请确认处理方式：**
> - **方案 A（多国共用）**：所有使用该 task/controller 的国家都在配置表中补齐所需配置项
> - **方案 B（隔离）**：将文件迁移到 `app/modules/{Country}/tasks/`（或 controllers）下，彻底与其他国家隔离

> BLL 文件（`app/BLL/`）无需此步骤，影响面可通过 grep 调用方自行判断。
