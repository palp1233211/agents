## git操作规范

### 分支名称
1. 分支名称格式：feature/[上线日期]/[作者名称]_[需求编号]
2. 上线日期 获取下周星期四的Ymd
3. 作者名称 作者名称首字母
4. 需求编号 根据提供的需求文档中获取，如果没有需求则根据内容创建一个不超过8个字母的英文描述

### 创建分支
1. 先检查当前分支状态，如果有待处理的事情，提示用户处理。
2. 分支必须根据master分支创建

### commit 备注格式
1. 必须以需求编号开头：`{需求编号} {type}: {描述}`
2. 示例：`23970 feat: 【PH】个人代理不接单提前通知天数改为配置表可调`
3. type 类型：feat / fix / refactor / docs / test / chore / perf

### 提交内容限制
1. **测试脚本禁止提交到仓库**（`tests/` 目录下所有文件不得 commit）
2. 只提交业务代码变更文件
