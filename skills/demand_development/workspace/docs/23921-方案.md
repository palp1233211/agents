## 三、接口变更（bi-common）

### 3.1 reviewAction — 审核接口

**文件：** `app/controllers/API/ParcelReweighAppealReviewController.php`

| 变更点 | 当前 | 变更后 |
|--------|------|--------|
| `review_size` 校验 | `in_array($val, [1,2])` 否则报错 | 允许 `0`，即 `in_array($val, [0,1,2])` |

**文件：** `app/BLL/ParcelReweighAppealReviewBLL.php` → `review()` 方法

| 变更点 | 逻辑 |
|--------|------|
| `review_size=0` 处理 | 计算 `review_result` 时将 `review_size=0` 视为 `1`（信任Flash）；`review_weight=0` 同理 |
| 落表 | `review_size` / `review_weight` 保持原值 `0` 写入 DB |
| MQ 推送 | `reviewSize` / `reviewWeight` 保持原值 `0` 推送 |

`review_result` 计算规则（引入 effective 变量）：

```php
$effective_weight = ($data['review_weight'] == 0) ? 1 : $data['review_weight'];
$effective_size   = ($data['review_size']   == 0) ? 1 : $data['review_size'];

// 用 $effective_weight / $effective_size 计算 review_result (1~4)
```

### 3.1.1 getListAdjustmentExcel — qaqc_weight / qaqc_size 计算修正

**文件：** `app/BLL/ParcelReweighAppealReviewBLL.php` → `getListAdjustmentExcel()`

当前逻辑 `review_size == 1` 才取 flash 值，`review_size=0` 会误取 client 值。需同步修正：

```php
// 修正前
$data[$k]['qaqc_weight'] = $datum['review_weight'] == 1 ? $datum['flash_parcel_weight'] : $datum['client_parcel_weight'];
$data[$k]['qaqc_size']   = $datum['review_size']   == 1 ? $datum['flash_parcel_size']   : $datum['client_parcel_size'];

// 修正后：0 视同 1（信任 Flash）
$data[$k]['qaqc_weight'] = ($datum['review_weight'] == 1 || $datum['review_weight'] == 0) ? $datum['flash_parcel_weight'] : $datum['client_parcel_weight'];
$data[$k]['qaqc_size']   = ($datum['review_size']   == 1 || $datum['review_size']   == 0) ? $datum['flash_parcel_size']   : $datum['client_parcel_size'];
```

### 10.5 BLL 核心逻辑

```php
class ReweighAppealReviewEmailBLL extends BaseBLL
{
    /**
     * 获取指定日期已审核数据，按 client_id 聚合
     * @param string|null $date 查询日期，默认昨天
     */
    public function getYesterdayReviewedData(?string $date = null): array
    {
        $targetDate = $date ?? custom_gmdate(strtotime('-1 day'), 'Y-m-d');
        $sql = "
            SELECT client_id, COUNT(*) as count, MIN(review_at) as review_at
            FROM fp_parcel_reweigh_appeal_review
            WHERE pno_appeal_type = 1
              AND DATE(review_at) = '{$targetDate}'
            GROUP BY client_id
        ";
        return $this->getDI()->get('db_bi_center_r')->query($sql)->fetchAll();
    }

    /**
     * 获取客户信息
     */
    public function getKaProfile(int $clientId): array
    {
        $sql = "
            SELECT id, customer_sub_group, department_id
            FROM ka_profile
            WHERE id = {$clientId}
        ";
        return $this->getDI()->get('db_rbi')->query($sql)->fetch();
    }

    /**
     * 获取配置邮箱（JSON层级结构查找）
     * @param string $customerSubGroup 客户分组
     * @param int $departmentId 部门ID
     * @return array
     * @throws \RuntimeException 配置不存在时抛异常
     */
    public function getCcEmails(string $customerSubGroup, int $departmentId): array
    {
        $emails = $this->buildCcEmailConfigKey($customerSubGroup, $departmentId);
        return explode(',', $emails);
    }

    /**
     * 层级查找抄送人邮箱（JSON配置）
     * 一级：department_id
     * 二级：customer_sub_group
     * 非数组即为最终值
     */
    private function buildCcEmailConfigKey(string $customerSubGroup, int $departmentId): string
    {
        $configJson = env('FP_APPEAL_REVIEW_EMAIL_CC_CONFIG');
        $config = json_decode($configJson, true);
        
        // 一级：部门ID
        if (!isset($config[$departmentId])) {
            return $this->getDefaultCcEmailConfig($config);
        }
        $level1 = $config[$departmentId];
        if (!is_array($level1)) {
            return $level1;  // 非数组即值
        }
        
        // 二级：客户分组
        if (!isset($level1[$customerSubGroup])) {
            return $this->getDefaultCcEmailConfig($level1);
        }
        $level2 = $level1[$customerSubGroup];
        if (!is_array($level2)) {
            return $level2;  // 非数组即值
        }
        
        return $this->getDefaultCcEmailConfig($level2);
    }

    /**
     * 发送审核结果邮件（模板从 .env 读取）
```
