## 5. 核心实现设计

### 5.1 AsyncBaseTask 子类

```
文件：app/modules/Th/tasks/async/ReweighAppealReviewEmailAsyncBaseTask.php
namespace：FlashExpress\bi\App\Modules\Th\Tasks
继承：\AsyncBaseTask

args_check()：
  - 校验 $this->args['bill_no'] 非空

handle()：
  1. $data = ReweighAppealReviewEmailBLL->getDataByBillNo(bill_no)
  2. if 空 || done_count < total_count → return true（不发邮件）
  3. 取 client_id、create_at
  4. getKaProfile(client_id)
  5. getRecipients(customer_sub_group, department_id)  ← 原 getCcEmails 逻辑
  6. if recipients 空 → log warning，return true
  7. sendReviewEmail(client_id, bill_no, create_at, recipients) → return bool
```

### 5.2 BLL 新增方法 getDataByBillNo

```sql
SELECT
    client_id,
    COUNT(*)                                                         AS total_count,
    SUM(CASE WHEN pno_appeal_type IN (2,3,4,5) THEN 1 ELSE 0 END)  AS done_count,
    MAX(create_at)                                                   AS create_at
FROM fp_parcel_reweigh_appeal_review
WHERE bill_no = '{bill_no}'
GROUP BY bill_no, client_id
```

返回单行 `[client_id, total_count, done_count, create_at]`，空则返回 `[]`。

### 5.3 BLL 改造 sendReviewEmail

- 签名去掉 `$ccEmails` 参数：`sendReviewEmail(string $clientId, string $billNo, string $createAt, array $recipients): bool`
- 标题模板：`{clientId} {billNo} {date('Ymd', strtotime($createAt))} 请及时帮助客户调账 / Please handle bills for customer timely`
- `Mail::send($recipients, $title, $content)` 不传 cc

### 5.4 收件人获取

直接复用 `getCcEmails($customerSubGroup, $departmentId)` 的层级查找逻辑，返回值即为收件人列表，方法本身不改名、不改逻辑。

## 方案自我校验

### 性能校验
- 查询复杂度：O(1)（按 bill_no 点查，已有索引 idx_review_at；bill_no 若无索引需确认）
- N+1 风险：无，handle 内最多 3 次 DB 查询（getDataByBillNo / getKaProfile / 无）
- 分页：单条记录，无需分页

### 内存校验
- 无大数据加载风险，仅返回单行聚合 + 一行 ka_profile

### 并发与一致性
- 幂等：bi-common 保证同一 bill_no 只创建一个 async task；即便重复触发，done_count==total_count 仍通过，发两封邮件风险存在
- 竞态风险：**有**——若审核进行中触发，total_count 可能包含未完成运单，done_count < total_count 则直接 return 不发，安全
- 失败重试：框架 flag=3 失败后由 bi-common 判断是否重试

### 高风险清单

| 风险点 | 影响面 | 处理动作 |
|--------|--------|----------|
| bi-common 重复触发同一 bill_no | 重复发邮件 | bi-common 侧保证幂等（创建 task 前校验已存在） |
| bill_no 在表中无索引，全表扫描 | 查询慢 | 确认 fp_parcel_reweigh_appeal_review 是否有 bill_no 索引，必要时补建 |
| getCcEmails 配置缺失抛异常 | 任务 flag=3 | 框架 catch 后标记失败，不影响审核流程 |
