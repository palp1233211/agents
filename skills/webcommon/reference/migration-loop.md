# Step 2：迁移循环模板

## 铁律重申

> **只复制，不改写。逻辑必须一致。不怕多，就怕错。**
>
> 迁移 = 原样复制 + 更新引用路径。禁止任何逻辑修改。

## 每个子步骤的三阶段循环

每个子步骤处理**一个文件**的迁移，走以下三阶段：

---

### 阶段 0：探索（Explore）

**必须使用 `Agent(subagent_type: Explore)`**，不在主上下文加载代码。

探索内容：
1. 读取要迁移的 webcommon 源文件完整内容
2. 确认文件的命名空间、类名、方法列表
3. 查找项目中所有引用该文件的位置（grep use/require/extends/implements）
4. 检查该文件是否依赖其他尚未迁移的 webcommon 文件

**产出模板**：

```
## 子步骤 {N} - 阶段 0 探索结果

### 源文件信息
- 路径：{webcommon源文件路径}
- 命名空间：{当前namespace}
- 类/接口/Trait 名：{名称}
- 行数：{行数}
- 依赖的 webcommon 文件：{列表，或"无"}

### 项目中的引用点
| 文件 | 行号 | 引用方式 | 引用内容 |
|------|------|---------|---------|
| {文件} | {行} | use/require/extends | {具体语句} |

### 迁移前置检查
- [ ] 该文件依赖的其他 webcommon 文件已迁移？
- [ ] 目标路径的父目录存在？
```

---

### 阶段 1：计划（Plan）

基于阶段 0 的探索结果，制定本子步骤的操作计划。

**产出模板**：

```
## 子步骤 {N} - 阶段 1 迁移计划

### 操作清单
1. 复制文件：`{源路径}` → `{目标路径}`
2. 命名空间调整：`{旧namespace}` → `{新namespace}`（或：无需调整）
3. 引用更新：
   - `{文件A}:{行号}` — `{旧引用}` → `{新引用}`
   - `{文件B}:{行号}` — `{旧引用}` → `{新引用}`

### 不做的事（明确排除）
- 不修改任何业务逻辑
- 不优化代码
- 不调整方法签名
- 不添加/删除方法

### 风险点
- {如有风险说明}
```

**等待用户"通过"后才进入阶段 2。**

---

### 阶段 2：执行 + 验证（Execute & Verify）

用户"通过"阶段 1 后，执行迁移操作。

**执行步骤**：

1. **复制文件**：将源文件内容原样写入目标路径
2. **调整命名空间**（仅 namespace 声明行，如需要）
3. **更新引用**：修改所有引用点的 use/require/include 指向新路径
4. **执行自检**

**自检清单**：

```
## 子步骤 {N} - 阶段 2 执行结果

### 变更摘要
| 操作 | 文件 | 说明 |
|------|------|------|
| 新增 | `{目标路径}` | 从 `{源路径}` 复制 |
| 修改 | `{引用文件}` | 更新 use/require 指向 |

### 自检清单
- [ ] 目标文件已创建，内容与源文件一致（仅 namespace 声明不同）
- [ ] 所有引用点已更新（共 {N} 处）
- [ ] 未修改任何业务逻辑代码
- [ ] 未添加/删除任何方法或属性
- [ ] diff 对比仅显示 namespace 和引用路径的变更

### 逻辑一致性验证
源文件与目标文件 diff（预期仅 namespace 行有差异）：
{列出差异行，确认全部是 namespace/use 调整}
```

**等待用户"通过"后，更新 STATE 进入下一个子步骤。**

---

## 子步骤间的状态流转

```
substep_1 阶段0 → 阶段1 → 用户通过 → 阶段2 → 用户通过
  ↓ STATE: start_step_2_substep_2
substep_2 阶段0 → 阶段1 → 用户通过 → 阶段2 → 用户通过
  ↓ STATE: start_step_2_substep_3
  ...
substep_N 阶段0 → 阶段1 → 用户通过 → 阶段2 → 用户通过
  ↓ STATE: start_step_3
```

## 异常处理

| 情况 | 处理方式 |
|------|---------|
| 发现源文件依赖未迁移的 webcommon 文件 | 停止当前子步骤，报告给用户，可能需要调整迁移顺序 |
| 目标路径已存在同名文件 | 停止，报告给用户，等待决策 |
| 引用更新后发现还有遗漏的引用点 | 补充更新，重新自检 |
| 用户对阶段 1 计划不通过 | 根据用户意见修改计划，重新提交 |
| 用户对阶段 2 结果不通过 | 回滚变更，根据用户意见重新执行 |
